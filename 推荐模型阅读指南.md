# 推荐系统算法模型阅读指南

本指南按时间线与技术脉络梳理主流推荐/CTR 模型，便于按需选读与对比学习。

---

## 一、总览与阅读路径


| 阶段      | 代表模型                     | 建议阅读顺序           | 关注点                |
| ------- | ------------------------ | ---------------- | ------------------ |
| 传统/线性   | LR                       | 1                | 基准与可解释性            |
| 因子分解    | FM → FFM                 | 2 → 3            | 稀疏下的二阶交互           |
| 记忆+泛化   | Wide&Deep → DeepFM → DCN | 4 → 5 → 6        | 线性+DNN、自动特征、显式高阶   |
| 兴趣建模    | DIN → DIEN               | 7 → 8            | 局部激活、序列兴趣演化        |
| 多任务/长序列 | ESMM、PLE、BST、SIM         | 9 → 10 → 11 → 12 | 样本偏差、负迁移、长序依赖、超长序列 |


---

## 二、按模型分章阅读

### 1. LR（逻辑回归）— <2010


| 维度       | 说明                |
| -------- | ----------------- |
| **核心机制** | 线性加权              |
| **解决痛点** | 作为基准模型，强调可解释性与稳定性 |
| **复杂度**  | 低                 |
| **典型应用** | 早期广告系统            |


**阅读建议**：先建立「线性 + 可解释」的基线认知，后续模型多在此之上做特征交互或深度扩展。

---

### 2. FM（因子分解机）— 2010


| 维度       | 说明             |
| -------- | -------------- |
| **核心机制** | 隐向量点积交互        |
| **解决痛点** | 稀疏数据下的二阶特征组合泛化 |
| **复杂度**  | O(kn)          |
| **典型应用** | 中小规模推荐         |


**阅读建议**：理解「隐向量 + 点积」如何替代显式二阶组合，解决稀疏与泛化问题，是后续 FM 系列（DeepFM 等）的基础。

---

### 3. FFM（场感知因子分解机）— 2016


| 维度       | 说明                    |
| -------- | --------------------- |
| **核心机制** | 场感知隐向量（同一特征在不同域用不同向量） |
| **解决痛点** | 细粒度的特征域（Field）交互差异    |
| **复杂度**  | O(n²k)                |
| **典型应用** | Kaggle 竞赛、Criteo 等    |


**阅读建议**：在 FM 基础上理解「Field」概念，以及为何需要为「特征×域」维护不同隐向量；可与 FM 对比参数量与效果。

---

### 4. Wide & Deep — 2016


| 维度       | 说明                          |
| -------- | --------------------------- |
| **核心机制** | Linear（Wide）+ DNN（Deep）联合训练 |
| **解决痛点** | 记忆（规则/历史共现）与泛化（稠密表征）的平衡     |
| **复杂度**  | 中                           |
| **典型应用** | Google Play                 |


**阅读建议**：重点理解「记忆 vs 泛化」的设计动机，以及 Wide 侧依赖人工交叉特征的局限，为 DeepFM/DCN 的改进做铺垫。

---

### 5. DeepFM — 2017


| 维度       | 说明                    |
| -------- | --------------------- |
| **核心机制** | FM + DNN，共享 Embedding |
| **解决痛点** | 自动特征组合，无需人工设计 Wide 特征 |
| **复杂度**  | 中                     |
| **典型应用** | 华为应用市场                |


**阅读建议**：对比 Wide&Deep，理解「用 FM 替代 Wide 人工特征」带来的自动化与效果提升；掌握共享 Embed 的架构图。

---

### 6. DCN（Deep & Cross Network）— 2017


| 维度       | 说明                   |
| -------- | -------------------- |
| **核心机制** | Cross Network 显式高阶交叉 |
| **解决痛点** | 显式、有界阶数的高阶特征组合       |
| **复杂度**  | 低（线性层堆叠）             |
| **典型应用** | 广告点击预估               |


**阅读建议**：理解 Cross 层如何用线性复杂度实现可控阶数的显式交叉，与 FM 的隐式、二阶形成对比。

---

### 7. DIN（Deep Interest Network）— 2018


| 维度       | 说明                                           |
| -------- | -------------------------------------------- |
| **核心机制** | Local Activation（Attention）：根据候选 item 激活历史行为 |
| **解决痛点** | 用户兴趣的多样性与局部相关性（一次只关注与当前候选相关的行为）              |
| **复杂度**  | 高（Attention）                                 |
| **典型应用** | 阿里妈妈、电商推荐                                    |


**阅读建议**：掌握「用户行为序列 + 候选 item → Attention 加权」的范式，这是后续 DIEN/BST/SIM 等序列模型的基础。

---

### 8. DIEN（Deep Interest Evolution Network）— 2019


| 维度       | 说明                         |
| -------- | -------------------------- |
| **核心机制** | GRU + Auxiliary Loss（兴趣演化） |
| **解决痛点** | 用户兴趣随时间的动态演化建模             |
| **复杂度**  | 极高（序列串行）                   |
| **典型应用** | 淘宝猜你喜欢                     |


**阅读建议**：在 DIN 的「局部激活」之上，理解「兴趣演化」：用 GRU 建模行为序列中的时序依赖，并配合辅助损失稳定训练。

---

### 9. ESMM（Entire Space Multi-Task Model）— 2018


| 维度       | 说明                                 |
| -------- | ---------------------------------- |
| **核心机制** | Multi-Task：CTR + CTCVR（或 CVR 相关目标） |
| **解决痛点** | CVR 预估中的样本偏差与稀疏性（点击未转化样本的利用）       |
| **复杂度**  | 中                                  |
| **典型应用** | 淘宝转化预估                             |


**阅读建议**：理解「全空间」建模：用 CTR、CTCVR 等任务共享表示、缓解 CVR 的样本选择偏差与稀疏，适合转化/下单等后续环节。

---

### 10. BST（Behavior Sequence Transformer）— 2019


| 维度       | 说明                                  |
| -------- | ----------------------------------- |
| **核心机制** | Transformer（Self-Attention）         |
| **解决痛点** | 利用 Self-Attention 捕获长程依赖，替代 RNN/GRU |
| **复杂度**  | 高                                   |
| **典型应用** | 淘宝搜索/推荐                             |


**阅读建议**：与 DIEN 对比：从「序列串行（GRU）」到「并行 Self-Attention」，理解长序列建模中 Transformer 的优势与计算成本。

---

### 11. SIM（Search-based Interest Model）— 2020


| 维度       | 说明                                               |
| -------- | ------------------------------------------------ |
| **核心机制** | GSU（General Search Unit）+ ESU（Exact Search Unit） |
| **解决痛点** | 超长（终身）用户行为序列的建模与检索                               |
| **复杂度**  | 极高（需索引与检索支持）                                     |
| **典型应用** | 阿里展示广告                                           |


**阅读建议**：理解「长序列」的工程与算法结合：如何通过检索（GSU/ESU）从终身行为中筛出与当前候选相关的子序列，再送入精排。

---

### 12. PLE（Progressively Layered Extraction）— 2020


| 维度       | 说明                                                 |
| -------- | -------------------------------------------------- |
| **核心机制** | Explicit Shared / Task-Specific Experts（显式共享与独占专家） |
| **解决痛点** | 多任务学习中的负迁移与跷跷板效应                                   |
| **复杂度**  | 高                                                  |
| **典型应用** | 腾讯视频                                               |


**阅读建议**：对比 MMoE：理解「显式分层、共享与独占专家」如何缓解多任务冲突与跷跷板，适合多目标（点击、时长、完播等）场景。

---

## 三、按「问题」选读

- **稀疏与泛化**：FM → FFM  
- **记忆与泛化**：Wide&Deep → DeepFM  
- **高阶交叉**：DCN（显式）、DeepFM（隐式）  
- **用户兴趣与候选相关**：DIN → DIEN → BST  
- **超长序列**：BST → SIM  
- **多任务/多目标**：ESMM（CTR-CVR）、PLE（多目标平衡）

---

## 四、复杂度与工程取舍（速查）


| 复杂度 | 模型                               | 工程注意点                                     |
| --- | -------------------------------- | ----------------------------------------- |
| 低   | LR, DCN                          | 易上线、易解释，适合基线或资源受限                         |
| 中   | FM, FFM, Wide&Deep, DeepFM, ESMM | 常规特征与多任务，工程成熟                             |
| 高   | DIN, BST, PLE                    | Attention/Transformer/多专家，显存与 latency 需优化 |
| 极高  | DIEN, SIM                        | 序列串行、索引与检索，需专门架构与团队                       |


---

按上述顺序阅读可先建立「线性 → 二阶交互 → 深度 + 记忆/泛化 → 兴趣序列 → 多任务/长序列」的完整脉络，再按业务痛点（稀疏、长序列、多目标等）回头精读对应章节。
